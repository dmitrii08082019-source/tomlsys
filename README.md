# tomlsys — Система, где всё — TOML

**tomlsys** — это экспериментальная операционная система (или её концептуальный прототип), в которой **все системные файлы представлены в формате TOML**. Вместо бинарных файлов, сложных скриптов и разрозненных конфигураций — единый, читаемый человеком формат.

## 🚀 Идея проекта

В традиционных ОС системные файлы разбросаны по разным папкам и имеют разные форматы. `tomlsys` идёт другим путём:

- **Всё — TOML**. Загрузчик, ядро, настройки памяти, встроенные приложения — всё описывается в TOML-файлах.
- **Прозрачность**. Любой пользователь может открыть системный файл и понять, как он устроен.
- **Гибкость**. Изменение параметров в TOML-файле меняет поведение системы без перекомпиляции.

## 📁 Структура системы

Все системные файлы находятся в папке `/System/`:

| Файл | Назначение |
|------|------------|
| `boot.toml` | Параметры загрузчика системы |
| `kernel.toml` | Настройки ядра (память, драйверы, процессы) |
| `notepad.toml` | Конфигурация встроенного текстового редактора |
| `tomlload.toml` | **Главный загрузчик** — диспетчер, который читает и применяет все остальные TOML-конфиги |
| `web.toml` | Настройки веб-интерфейса или встроенного веб-сервера |
| `ramfull.toml` | Конфигурация поведения системы при **100% заполнении** оперативной памяти |
| `rammemoryminium.toml` | Конфигурация для **режима экономии памяти** (при критически низком уровне RAM) |

## ⚙️ Как это работает

1. Система запускается с `tomlload.toml` — это главный диспетчер.
2. `tomlload.toml` читает остальные конфиги и применяет их.
3. В зависимости от состояния памяти (`ramfull.toml` или `rammemoryminium.toml`) система может динамически менять своё поведение.
4. Все компоненты (ядро, веб, редактор) управляются через TOML.
